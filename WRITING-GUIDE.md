# 博客文章写作指南

## 📋 标准模板位置

`_templates/article-template.md`

## 🎯 模板结构说明

### 1. Frontmatter（文章元数据）
```yaml
---
layout: post
title: "文章标题"           # 简洁有力，带问号或数字更佳
date: 2026-03-09T00:00:00+08:00
tags: [AI-Native软件工程, 标签1, 标签2]
author: Sophi
series: AI-Native软件工程系列 #XX  # 系列编号
---
```

### 2. TL;DR（极其重要）
- 4点核心观点
- 每点格式：**观点** — 简短描述
- 让读者30秒抓住价值

### 3. 本文结构（目录导航）
- 6个章节锚点
- 每节配简短副标题
- 支持跳读

### 4. Key Insight 卡片
使用引用格式：
```markdown
> 💡 **Key Insight**
> 
> 核心观点一句话总结
```

### 5. 段落长度
- **每段2-4行**
- 避免大段文字
- 适当使用空行分隔

### 6. 表格使用
- 对比表格（传统 vs AI-Native）
- 时间线表格（历史演进）
- 最多4-5列，避免过宽

### 7. ASCII 图表
```
┌─────────────────┐
│  Layer 4: 顶层   │
├─────────────────┤
│  Layer 3: 中层   │
├─────────────────┤
│  Layer 2: 下层   │
├─────────────────┤
│  Layer 1: 基础   │
└─────────────────┘
```

### 8. 路线图清单
```markdown
**阶段一：准备（1-2个月）**
- [ ] 任务1
- [ ] 任务2
```

### 9. Takeaway 对比表
| 传统思维 | AI-Native思维 |
|---------|--------------|
| 旧观点 | 新观点 |

### 10. 延伸阅读
- 经典案例（3个）
- 本系列相关（2-3篇）
- 学术理论（2-3本）

## ✅ 写作检查清单

发布前检查：

- [ ] 标题是否有吸引力？
- [ ] TL;DR 是否完整？
- [ ] Key Insight 是否突出？
- [ ] 段落是否简短（2-4行）？
- [ ] 表格是否清晰？
- [ ] 是否有 ASCII 图或对比表？
- [ ] Takeaway 是否总结到位？
- [ ] 延伸阅读是否相关？
