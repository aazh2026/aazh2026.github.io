# 博客写作标准操作程序 (SOP)

> **每次写博客前，打开此文档检查一遍**
> 
> 目标：确保文章符合"思想传播友好型"标准

---

## 🚀 快速检查清单 (30秒)

发布前必须检查：

- [ ] **TL;DR** 有4点核心观点？
- [ ] **结构导航** 有6个章节锚点？
- [ ] **段落** 每段不超过4行？
- [ ] **Key Insight** 每章至少1个？
- [ ] **对比表格** 有传统 vs AI-Native？
- [ ] **Takeaway** 结尾有对比总结？

**全勾才能发布！**

## ⚠️ 常见错误清单

### ❌ 绝对不要做的事

| 错误 | 正确做法 |
|------|---------|
| 段落超过6行 | 控制在2-4行 |
| 纯文字无表格 | 至少1个对比表格 |
| 无TL;DR | 必须有4点TL;DR |
| 无结构导航 | 6章节锚点导航 |
| 标题用过去时 | 用现在时或问句 |
| 日期时间随意 | 统一用 `T00:00:00+08:00` |
| 链接到外部无描述 | 简要说明链接内容 |

## 🔧 技术发布流程

### 发布前检查

1. **本地预览**（如可能）
   ```bash
   bundle exec jekyll serve
   ```

2. **YAML检查**
   - 标题无嵌套引号
   - 日期格式正确
   - tags用方括号

3. **Push后验证**
   - 等待2-3分钟构建
   - 检查文章链接可访问
   - 检查TOC显示正常
   - 检查移动端显示

4. **发布后分享**
   - 复制Key Insight到Twitter
   - 分享文章链接

## 💡 写作原则速记

### 5秒原则
读者5秒内决定离开还是继续。
- TL;DR 必须在前3行
- 结构导航清晰可见

### 扫描原则
80%读者只扫描不细读。
- 多用表格
- 多用列表
- 多用加粗

### 传播原则
好内容要易于分享。
- Key Insight 一句话
- 可截图的金句
- 对比表一目了然
