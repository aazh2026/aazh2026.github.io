# MEMORY.md - 长期记忆

## 博客写作范式转移（2026-03-09）

### 核心转变

**从"论文风格"到"博客风格"**

之前：文章像学术论文，大段文字，线性阅读，难以扫描
之后：文章像Twitter线程，结构化，扫描友好，易于传播

### 关键学习

**读者行为**
- 80%读者只扫描20%内容
- 平均阅读时间决定文章成败
- 金句和Key Insight决定传播力

**内容结构黄金法则**
1. TL;DR 必须有（4点核心观点）
2. 结构导航（6章节锚点）
3. 段落限制（2-4行）
4. Key Insight 卡片（每章至少1个）
5. 对比表格（传统 vs AI-Native）
6. Takeaway 总结（对比表）

**技术教训**
- Jekyll日期时间要用 `T00:00:00+08:00` 避免未来时间问题
- YAML frontmatter不能有嵌套引号
- GitHub Pages构建队列可能繁忙，需要耐心等待
- 文章宽度限制在680px最佳阅读体验

### 已建立的标准

**位置**：`BLOG-SOP.md`
**模板**：`_templates/article-template.md`
**指南**：`WRITING-GUIDE.md`

每次写博客前，必须阅读SOP并检查清单。
