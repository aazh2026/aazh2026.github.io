## 一、为什么需要结构化

Claude Code 101 解决了单篇论文或短期项目的问题。但学术研究者的项目往往持续数月甚至数年，积累数百篇论文和多个草稿。

如果你把这些全塞进一个文件夹、只用一个 CLAUDE.md，Claude 给你的结果会是同质化的——它无法针对你的具体工作给出精准定制化的输出。

这和人类研究助理的逻辑一样：你想让研究助理帮你起草论文某章节、清洗数据集、或标注一篇文章，你会给每个任务不同的指令集。

Claude Code 也能做到——通过创建子文件夹实现同样的组织逻辑。

## 二、嵌套 CLAUDE.md 架构

### Global CLAUDE.md（项目宪法）

放在项目根目录，定义整个项目的基本规则：

```markdown
# Who I Am
[你的身份和研究方向]

# Project Overview
[项目简介和研究目标]

# Writing Style
[你的写作风格偏好]

# Key Constraints
[研究伦理、数据处理等必须遵守的规则]

# Literature Database
[/literature/ 文件夹包含我所有已读的文献]
```

### Local CLAUDE.md（子文件夹专属指令）

在子文件夹内放置针对该任务的专属 CLAUDE.md：

```
/literature/
  → CLAUDE.md: 文献检索策略、引用格式规范

/drafts/
  → CLAUDE.md: 写作风格指南、章节结构要求

/datasets/
  → CLAUDE.md: 数据处理规范、清洗标准

/reviews/
  → CLAUDE.md: 审稿标准、同行评审协议
```

这样 Claude 进入每个子文件夹时，会自动加载该文件夹的专属上下文，不会被全局上下文稀释。

## 三、Plan Mode

对于复杂任务，不要让 Claude 直接执行——先让它出计划。

在 Claude Code 中输入 `/plan` 或在 prompt 中加 "think step by step"，Claude 会先列出执行步骤，你确认后再行动。

**为什么这很重要：**

1. **防止 context clutter** — 一次列出所有步骤比边做边改更高效
2. **提前发现逻辑错误** — 在执行前就能看到问题
3. **保持人工对齐** — AI 的计划可能不符合你的意图，确认机制是安全网

## 四、Custom Slash Commands

学术研究中有大量重复性任务：每次读新论文都要问"这篇论文的核心论点是什么"、每次清洗数据都要重复同样的处理流程。

把这些变成一行命令：

```markdown
/slashi 帮我总结这篇论文的核心贡献
/slash 清洗这个数据集
/slash 查找与我的研究假设相关的文献
/slash 检查我的引用格式是否规范
```

创建方式：在 `.claude/commands/` 目录下创建 `.md` 文件，每个文件就是一个 slash command。

## 五、Subagents

当一个任务太大、一个上下文窗口装不下时，用 Subagents。

Subagent 在独立的新上下文窗口中运行，不受主会话 context 污染。

**适用场景：**

- 你的项目有多个独立方向同时推进
- 某个任务需要长时间深度分析，会话快结束时还没做完
- 需要并行处理多个相似的子任务

**使用方法：**

```
/subagent
[描述子任务内容]
```

Claude 会创建一个新的独立会话处理这个任务，完成后把结果带回来。

## 六、学术研究专用 Subagents

Mushtaq Bilal 建议为学术研究定制四个专用 Subagent：

### 1. Literature Reviewer

- 职责：扫描最新文献，找出与研究问题相关的论文
- 触发条件：开始新主题研究时
- 输出：带注解的文献列表 + 研究空白分析

### 2. Citation Checker

- 职责：验证引用完整性——检查引用的页码、作者名、年份是否准确
- 触发条件：提交论文前、响应审稿意见时
- 输出：引用错误报告 + 修正建议

### 3. Methodology Auditor

- 职责：审查研究方法的逻辑一致性——样本量、变量选择、分析框架是否合理
- 触发条件：研究设计阶段、定稿前
- 输出：方法论问题清单 + 改进建议

### 4. Reviewer 2 Simulation

- 职责：模拟挑剔审稿人第二的视角，预判他会提什么问题
- 触发条件：投稿前自我评审时
- 输出：潜在审稿问题列表 + 应对策略

## 七、延伸思考

Claude Code 101 → 102 → 103... 这个系列本质上演示的是**如何把 AI 工具变成认知伙伴**的问题，而不是工具使用教程。

嵌套 CLAUDE.md 的设计原则值得所有长期知识工作借鉴：**不同层次的上下文需要不同层次的指令**。全局宪法定义方向，局部指令定义执行细节。这和 Garry Tan 的 skillify 有相通之处——不是一次性的 prompt，而是持续演进的系统。

Subagents 的概念解决了 AI 辅助研究的一个根本矛盾：深度和广度之间的注意力取舍。一个 agent 深入一个问题，另一个 agent 平行处理另一个，它们各自保持独立上下文，最终汇总——这是真正意义上的人类思维扩展，而不是简单的任务并行。
