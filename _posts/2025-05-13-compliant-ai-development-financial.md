## 金融AI合规的监管背景

### 全球金融监管趋势

**欧盟**：
- **EU AI Act**：全球首个全面AI监管法规
- 高风险AI系统（包括信贷评估、保险定价）需满足严格要求
- 要求：可解释性、人工监督、数据质量、透明度

**美国**：
- **OCC、Fed、FDIC**联合发布AI风险管理指南
- 要求：模型风险管理（MRM）、公平借贷、消费者保护
- 特别关注：算法歧视、模型漂移

**中国**：
- **《生成式人工智能服务管理暂行办法》**
- **金融AI应用规范**：要求可解释、可审计、安全可控
- 数据本地化要求：金融数据不得出境

**新加坡**：
- **FEAT原则**（Fairness, Ethics, Accountability, Transparency）
- 行业自律 + 监管指导的柔性监管模式

### 金融监管的核心关切

**关切1：算法公平性**

监管问题：
- AI系统是否对某些群体存在歧视？
- 信贷审批是否因种族、性别而区别对待？
- 保险定价是否对特定人群不公平？

**关切2：可解释性**

监管要求：
- 拒绝贷款申请时，必须说明理由
- AI做出的风险评级，必须能解释依据
- 监管检查时需要理解模型决策逻辑

**关切3：稳定性与鲁棒性**

监管要求：
- AI系统不能因数据漂移而突然失效
- 必须有应对极端情况的机制
- 模型更新需要严格的验证流程

**关切4：数据隐私**

监管要求：
- 客户数据的使用必须获得明确授权
- 数据最小化原则：只收集必要数据
- 数据安全：防止泄露和滥用

## AI风险控制机制

### 偏见检测与缓解

**检测方法**：

**方法1：统计指标检测**
```python
def demographic_parity_check(y_pred, sensitive_attr):
    """
    人口统计平等：各群体的正例预测率应该相等
    """
    groups = sensitive_attr.unique()
    rates = {group: y_pred[sensitive_attr == group].mean() 
             for group in groups}
    max_diff = max(rates.values()) - min(rates.values())
    return max_diff < THRESHOLD, rates

def equal_opportunity_check(y_true, y_pred, sensitive_attr):
    """
    机会平等：各群体的真正例率(TPR)应该相等
    """
    groups = sensitive_attr.unique()
    tprs = {}
    for group in groups:
        mask = sensitive_attr == group
        tp = ((y_true[mask] == 1) & (y_pred[mask] == 1)).sum()
        fn = ((y_true[mask] == 1) & (y_pred[mask] == 0)).sum()
        tprs[group] = tp / (tp + fn) if (tp + fn) > 0 else 0
    max_diff = max(tprs.values()) - min(tprs.values())
    return max_diff < THRESHOLD, tprs
```

**方法2：因果推断检测**
```python
# 使用因果图检测直接和间接歧视
def causal_fairness_check(model, data, sensitive_attr, outcome):
    """
    检测敏感属性对结果的因果影响
    """
    # 构建因果图
    causal_graph = build_causal_graph(data, sensitive_attr, outcome)
    
    # 计算直接歧视
    direct_effect = estimate_direct_effect(causal_graph, sensitive_attr, outcome)
    
    # 计算间接歧视（通过中介变量）
    indirect_effects = estimate_indirect_effects(causal_graph, sensitive_attr, outcome)
    
    return direct_effect, indirect_effects
```

**缓解策略**：

**策略1：预处理（Pre-processing）**
- 数据重采样：平衡不同群体的样本
- 数据变换：消除敏感属性的影响
- 合成数据：生成公平的数据集

**策略2：处理中（In-processing）**
- 公平性约束：在训练时加入公平性损失函数
- 对抗训练：训练模型消除对敏感属性的依赖
- 多任务学习：同时优化性能和公平性

**策略3：后处理（Post-processing）**
- 阈值调整：不同群体使用不同决策阈值
- 校准：调整模型输出使其在不同群体间一致
- 拒绝选项：对边界案例进行人工审查

### 模型风险管理（MRM）

**风险分类**：

| 风险类型 | 描述 | 控制措施 |
|---------|------|---------|
| **模型风险** | 模型预测错误导致的损失 | 监控、验证、回滚机制 |
| **数据风险** | 数据质量、偏见、隐私问题 | 数据治理、质量检查 |
| **操作风险** | 系统故障、人为错误 | 自动化、审计、培训 |
| **合规风险** | 违反监管要求 | 合规检查、法律咨询 |
| **声誉风险** | AI错误导致的声誉损失 | 公关预案、透明沟通 |

**三道防线模型**：

```
第一道防线：业务部门
- 模型开发和维护
- 日常监控
- 初步风险评估

第二道防线：风险管理/合规部门
- 独立模型验证
- 合规审查
- 风险监控

### 人工复核机制

**自动触发人工复核的场景**：

1. **高风险决策**
   - 大额交易审批（>100万）
   - 关键客户拒绝
   - 异常情况标记

2. **模型置信度低**
   - 预测置信度 < 70%
   - 多个模型结果不一致
   - 输入数据异常

3. **公平性预警**
   - 某群体通过率异常低
   - 公平性指标接近阈值
   - 用户投诉涉及歧视

4. **监管要求**
   - 新客户首次审批
   - 监管抽查样本
   - 审计期间

**复核流程**：
```
AI决策 → 触发条件检查 → 人工复核队列
                              ↓

                              ↓
记录复核结果和理由
                              ↓

## 数据隐私与合规

### 数据使用合规框架

**数据分类**：

| 分类 | 示例 | 处理要求 |
|------|------|---------|
| **公开数据** | 公开的统计信息 | 自由使用 |
| **内部数据** | 匿名化的交易记录 | 内部使用，需审批 |
| **敏感数据** | 个人身份信息(PII) | 脱敏后使用，严格授权 |
| **机密数据** | 客户完整档案 | 禁止用于AI训练 |

**数据处理原则**：

1. **目的限定**
   - 数据只能用于声明的目的
   - 禁止数据用途的随意扩展
   - 定期审查数据使用情况

2. **数据最小化**
   - 只收集必要的数据
   - 训练模型时排除敏感字段
   - 及时删除不再需要的数据

3. **透明度**
   - 向用户说明数据使用方式
   - 提供数据访问和删除途径
   - 定期发布数据使用报告

4. **安全保障**
   - 数据加密（传输和存储）
   - 访问控制（最小权限原则）
   - 定期安全审计

### 隐私保护技术

**技术1：差分隐私（Differential Privacy）**
```python
import opendp

# 在数据查询时添加噪声，保护个体隐私
def private_mean_query(data, epsilon=0.1):
    """
    epsilon越小，隐私保护越强，但数据效用越低
    """
    # 添加拉普拉斯噪声
    noisy_mean = laplace_mechanism(data.mean(), epsilon)
    return noisy_mean
```

**技术2：联邦学习（Federated Learning）**
```
本地数据不离开本地
    ↓
各客户端训练本地模型
    ↓
只上传模型参数（不上传原始数据）
    ↓
中央聚合参数，更新全局模型
    ↓
下发全局模型到各客户端
```

**技术3：同态加密（Homomorphic Encryption）**
- 数据在加密状态下进行计算
- 计算结果解密后与明文计算结果一致
- 适用于云计算场景

**技术4：数据脱敏**
```python
# 姓名脱敏
def mask_name(name):
    return name[0] + "**"  # 张三 → 张**

# 身份证号脱敏
def mask_id(id_number):
    return id_number[:6] + "********" + id_number[-4:]

# 数据泛化
def generalize_age(age):
    return (age // 10) * 10  # 35 → 30-40
```

**洞察2：合规需要技术、法律、业务的协同**

单靠技术团队无法实现合规，需要：
- 技术：实现可解释性、可追溯性
- 法律：理解监管要求
- 业务：平衡合规与业务目标

**洞察3：自动化是合规的关键**

人工合规检查无法应对AI系统的规模和速度，必须建立自动化合规工具链。

**洞察4：合规是动态的，不是静态的**

- 监管法规在演进
- AI技术在进步
- 合规实践需要持续更新
