Matt Van Horn 上周发了一篇文章，标题是"Every Agentic Engineering Hack I Know"。613K 阅读量，配套的开源项目加起来 27K+ stars。

他做的事听起来不复杂：用 Claude Code 和 Codex 两个人工智能引擎并行工作，每天开四到六个 session，同时处理不同的任务。但他描述的工作状态——以及支撑这个状态的具体技巧——值得认真拆解，因为它指向了一个正在发生的重要转变。

## 不再是"和 AI 对话"，而是"让 AI 协作"

传统的 AI 使用模式是对话式的：你问一个问题，AI 给一个回答，你继续追问。

Matt 的工作模式不是这样。他的工作流是这样的：

- 一个 session 在写 plan.md
- 一个 session 在用另一个已经写好的 plan 构建功能
- 一个 session 在运行 last30days 统计自己过去一个月的产出
- 一个 session 在修复测试中发现的 bug

他几乎不读 plan.md。他的原话是："Plans are for agents, you silly human."

这句话听起来像是调侃，但背后有一个真实的认知转变：**当你不再需要自己读计划的时候，计划的存在意义就变了。**

计划不再是你理解任务的方式，而是你控制 agent 行为的方式。你用写计划这个动作来确保 AI 不会偷懒、不会停在半成品、不会用最省力的方式交差。计划是一个约束装置，不是一个信息传递装置。

## "Make a plan for the plan" 为什么有效

Matt 最反直觉的一个技巧是：当他要做真正深度的工作时，他不让 AI 直接写文档或做分析，而是先让 AI 写一个"怎么读这些材料、怎么生产这个文档的计划"。

他的原话：ask it to first plan how it will produce the deliverable, then execute that plan, and it does the deep version every time.

这背后的机制是什么？

当 AI 被要求直接给出一个深度答案时，它倾向于用最通用的方式组织内容——因为那样 tokens 最省，上下文窗口最干净，风险最低。但这产生的是一份"差不多正确"的报告。

当 AI 被要求先计划怎么做，它必须在内心先模拟一个更高质量的结果会是什么样的，然后倒推出一个执行路径。这个模拟过程本身就是深度推理，而不仅仅是执行。

换句话说：**要求 AI 先计划，相当于强迫它先想清楚"什么才是一个真正好的答案"，而不是直接跳到"什么是最快的答案"。**

这个技巧不需要任何特殊的 prompt engineering。它只需要你改变请求的结构——从"给我 X"变成"先告诉我你会怎么做 X，然后做 X"。

## YOLO 模式的逻辑

Matt 的配置文件里有一行：

```
skipDangerousModePermissionPrompt: true
```

他开了六个并发 session。如果每个 session 都要等他点确认，他就是在当人肉哨兵。六个 AI 同时工作，每一个都在等他的批准——这比不用 AI 还累。

所以他选择了 YOLO 模式：默认允许所有操作，出问题了 git revert。他的安全网是 GitHub，不是逐个确认对话框。

这个选择有一个前提：**他相信自己的代码有版本控制，有备份，有方式恢复。** 如果你没有一个可靠的 git 恢复机制，YOLO 模式就是一个灾难。

但如果你有——那逐个确认对话框就是最大的效率瓶颈。

## 双引擎架构

Matt 实际上在使用两个 $200/月的订阅：Claude Max 和 Codex Max。Claude 负责规划、研究、 taste 类的判断；Codex 负责构建、实现、快速的代码生成。

他把这两个引擎当成两个不同的 worker，而不是两个不同的工具。Codex 速度快、便宜，接到指令就执行；Claude 推理能力强，处理模糊的、需要判断的任务。两个引擎通过 /ce-work --codex 这个命令桥接——从 Claude 的 planning session 里直接调起 Codex 的构建能力，不需要切换窗口，不需要复制粘贴。

这个架构有意思的地方在于：它说明"哪个模型"已经不再是一个用户的决策，而是一个工作流的内部调度决策。Matt 本人在选择用什么模型做什么事上花的精力微乎其微——大部分由 /ce-work 和 plugin 框架来处理。

## 一个值得关注的问题

Matt 描述的状态是：他去年一个人用这套工作流发布了三个产品，其中一个 last30days 是 27K stars 的 skill，还有一个 Printing Press 是 4K+ stars。

他上一次"真正发布过有人用的软件"是什么时候？他的回答是：高中毕业以后。

这是一个关于 AI 编程工具最值得注意的数据点：它让一个人重新变成了能够独立交付产品的开发者。不是因为 AI 突然变强了，而是因为 AI 把整个过程中最耗费精力的部分——规划、调试、多任务切换——变得可以委托了。

这和工业革命时期纺织机械的意义类似：不是机器比工人更能干，而是机器让一个人能完成以前需要很多人协作才能完成的工作。个体的生产力杠杆，在某些条件下，可以发生数量级的变化。

Agentic Engineering 正在证明这件事。
